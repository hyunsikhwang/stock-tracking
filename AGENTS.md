# Custom Instructions for Antigravity Agent

## 1. 언어 및 문서화 원칙
- **기본 언어:** 모든 사고 과정, 실행 계획(Plan), 코드 설명 및 문서는 반드시 **한국어**로만 작성합니다.
- **문서화:** 새로 생성되거나 수정되는 모든 문서는 한국어 맞춤법과 기술 용어를 준수하여 작성합니다.

## 2. 코드 수정 후 자동화 워크플로우 (Automated Workflow)
코드 수정 작업이 완료된 직후, 에이전트는 별도의 지시가 없어도 다음 단계를 **자동으로 수행**해야 합니다.

1. **상태 확인:** `git status`를 통해 변경 사항을 확인합니다.
2. **커밋 메시지 생성:** - 작업 내용을 요약하여 **한국어**로 커밋 메시지를 작성합니다. 
   - 형식 예시: `[수정] 특정 기능 로직 개선 및 버그 수정`
3. **스테이징 및 커밋:** - `git add .` 명령을 수행합니다.
   - 생성된 한국어 메시지로 `git commit -m "메시지"`를 실행합니다.
4. **원격 저장소 푸시:** - `git push`를 실행하여 원격 저장소에 반영합니다.

## 3. 예외 처리
- 만약 `git push` 과정에서 충돌(Conflict)이 발생할 경우, 작업을 중단하고 사용자에게 한국어로 상황을 보고한 뒤 지침을 기다립니다.
- 커밋할 내용이 없는 경우(No changes), 커밋 과정을 생략하고 사용자에게 알립니다.
