- 카드 UI 관련 수정은 동작 문제를 고치더라도 기존 시각 구조와 정보 밀도를 먼저 기준 커밋과 비교한 뒤 반영한다.
- 사용자가 특정 날짜 기준 UI 상태를 요구하면 그 날짜의 실제 커밋 출력과 동일한지 먼저 대조하고, 동작 차단은 스타일/이벤트 차단처럼 최소 변경으로 처리한다.
- Streamlit deprecation 경고를 처리할 때는 경고 문구만 믿지 말고 해당 버전의 실제 public API 시그니처와 런타임 렌더링 결과를 함께 검증한 뒤 교체한다.
- 사용자가 특정 차트/라이브러리 스택을 유지하라고 한 맥락에서는, 동작 문제 해결을 위해서도 대체 라이브러리로 갈아타지 말고 기존 스택 안에서 먼저 해결책을 찾는다.
